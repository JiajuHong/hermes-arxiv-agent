## 任务：每天自动监控 arXiv 论文并推送邮件

### 第一步：执行 Python 脚本
cd 到项目根目录后，用 python3 运行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py
```

### 第二步：判断是否有新论文
运行完毕后检查脚本输出：
- 如果包含 `No new papers`，直接回复：`✅ 今日（YYYY-MM-DD）未发现新的论文。`
- 如果有 `LLM_SUMMARIZATION_REQUIRED` 部分，进入第三步。

### 第三步：读取 JSON 并完成 LLM 信息补全
读取 `/path/to/hermes-arxiv-agent/new_papers.json`，该文件包含 `new_count`、`pending_count`、`new_papers` 和 `papers_to_process`。

其中：
- `new_count` 表示本次新抓取的论文数
- `pending_count` 表示当前仍需要 agent 补全的论文数
- 优先以 `papers_to_process` 作为本次必须处理的论文列表；若不存在，再使用 `new_papers`

对待处理列表中的每篇论文必须完成以下两步：

**1）提取作者单位 affiliations**
- 使用 pdfplumber 或 PyMuPDF 读取 PDF（路径在 pdf_local_path 字段）前 2 页文本
- 优先使用上标脚注映射（如 ^1、^2 对应单位）
- 如无清晰脚注，再匹配常见机构名（MIT、Stanford、Google、Microsoft、Tsinghua、Peking 等）
- 对 CamelCase 连写词做分词还原：`DepartmentofPoliticalSciences` → `Department of Political Sciences`
- 合并跨行连字符词：`Repub-` + `licof Korea` → `Republic of Korea`
- 删除噪音：URL、正文叙述句、公式片段、参考文献片段
- 若含连续 18+ 个无空格英文字母，必须再次清洗
- 多个单位用 `;` 分隔，不要换行，不要保留脚注编号
- 无法确定时填：`未找到单位信息`

**2）生成中文总结 summary_cn**
- 基于 abstract 生成，90-150 个中文字符
- 必须覆盖：方法核心、主要贡献、关键结果
- 禁止模板化空话，禁止仅写"提出新型方法"等泛化句
- 只输出一段自然中文，不使用分点
- 质量自检：少于 90 字或与其他摘要高度雷同时重写

### 第四步：更新 Excel
将 affiliations 和 summary_cn 回填到 `/path/to/hermes-arxiv-agent/papers_record.xlsx`（按 arxiv_id 匹配）。

### 第五步：更新网站数据
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 viewer/build_data.py
```

确认 `viewer/papers_data.json` 已根据最新 Excel 重建，包含刚回填的 affiliations 和 summary_cn。

### 第六步：同步待处理队列
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py --sync-pending-state
```

要求：
- 该步骤必须在 Excel 回填完成后执行
- 该步骤会根据最新 Excel 状态重建 `pending_llm_ids.txt`
- 若全部论文都已完成单位提取和中文总结，应自动清空 `pending_llm_ids.txt`
- 该步骤还会同步刷新 `new_papers.json`，避免 agent 重试时继续读取旧的待处理论文列表

### 第七步：发布静态网站
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && bash scripts/publish_viewer.sh
```

要求：
- 仅提交站点相关文件，不要提交 `papers_record.xlsx`、`papers/`、`viewer/favorites.json`
- 如果 `viewer/papers_data.json` 没有变化，可以跳过提交和 push
- push 到你自己的 GitHub fork 后会触发 GitHub Actions 更新 GitHub Pages
- 当前仓库必须是你自己的 fork，或 `PUBLISH_REMOTE` 必须明确指向你自己的可写远端

### 第八步：发送邮件推送

用 himalaya 发送 HTML 邮件到 `YOUR_EMAIL`。

**邮件格式要求：**

- 主题：`📚 每日论文推送 | YYYY-MM-DD`
- 收件人：`YOUR_EMAIL`
- 必须使用 MML multipart 格式，同时包含纯文本版和 HTML 版

MML 模板结构：
```
<#multipart type=alternative>
<#part type=text/plain>
📚 论文日报 | YYYY-MM-DD
共发现 N 篇新论文

[每篇：标题、arXiv ID、作者、单位、摘要]

<#part type=text/html>
<!DOCTYPE html>
<html>
<head><meta charset="utf-8"></head>
<body style="font-family: -apple-system, sans-serif; max-width: 680px; margin: 0 auto;">
<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); padding: 32px; text-align: center; border-radius: 12px 12px 0 0;">
  <h1 style="color: #fff; margin: 0; font-size: 24px;">📚 论文日报</h1>
  <p style="color: rgba(255,255,255,0.85); margin: 8px 0 0 0;">YYYY-MM-DD | 共发现 N 篇新论文</p>
</div>
[每篇论文一个卡片：标题链接、作者、单位、中文摘要、PDF按钮]
</body>
</html>
<#/multipart>
```

**论文卡片 HTML 格式（每篇）：**
```html
<div style="border-left: 4px solid #667eea; padding: 16px; margin: 16px 0; background: #f8f9fa; border-radius: 0 8px 8px 0;">
  <h3 style="margin: 0 0 4px 0;"><a href="https://arxiv.org/abs/ARXIV_ID" style="color: #333; text-decoration: none;">TITLE</a></h3>
  <p style="margin: 4px 0; color: #555; font-size: 13px;">👤 AUTHORS</p>
  <p style="margin: 4px 0; color: #555; font-size: 13px;">🏢 AFFILIATIONS</p>
  <p style="margin: 8px 0; color: #333; font-size: 14px; line-height: 1.6;">SUMMARY_CN</p>
  <a href="PDF_URL" style="display: inline-block; padding: 6px 16px; background: #667eea; color: #fff; text-decoration: none; border-radius: 6px; font-size: 13px;">📄 PDF</a>
</div>
```

**发送命令：**
```bash
cat << 'EMAIL_EOF' | himalaya template send --subject "📚 每日论文推送 | YYYY-MM-DD" --to YOUR_EMAIL
<MML_CONTENT>
EMAIL_EOF
```

**重要：如果今日无新论文且无待处理论文，也要发送一封邮件告知"今日未发现新论文"，不要不发邮件。**
**发送成功后回复确认信息给用户，不要输出邮件原文。**
